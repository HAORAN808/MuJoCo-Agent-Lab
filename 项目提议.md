# 面向机器人研发实验的主动实验设计与具身执行闭环智能体

## 具体目标

构建一个面向机器人研发实验的通用智能体原型，将"提出假设-设计实验-调用仿真-分析失败-建议下一轮实验"组织成可执行闭环。支持输入任意机械臂操作任务的自然语言描述，AI 自动完成任务解析、场景组装、实验设计、真实 MuJoCo 物理仿真、结果分析和下一轮迭代建议。

当前 Demo 支持 8 个 MuJoCo Menagerie 机械臂（FR3、Panda、UR5e、UR10e、Kinova Gen3、KUKA iiwa、xArm7、Lite6），产出包括实验变量空间、批量评测日志、失败归因、下一轮实验建议、轨迹回放与自动报告。

## 核心价值

机器人算法研发常依赖人工调参和经验复盘，实验记录难复用，失败边界难定位。本项目让智能体承担"研发实验员"角色：

- **通用任务解析**：输入自然语言描述，AI 自动解析为结构化任务规格（机器人、物体、动作、成功标准）
- **自动场景组装**：根据任务规格自动加载真实机械臂 MJCF 模型、物体和工作台
- **真实物理仿真**：使用 MuJoCo 物理引擎执行 Jacobian IK 控制、接触检测、摩擦和重力仿真
- **智能实验设计**：自动生成假设和变量扫描矩阵
- **失败归因**：解释在何种条件下实验失败，区分滑落、未接触、位移不足等失败类型
- **迭代优化**：基于历史结果收窄变量空间，检测收敛，给出下一轮建议

## 技术架构

```text
自然语言任务描述
  -> LLM 任务解析 (task_type, robot, objects, actions, success_criteria)
  -> 动态场景组装 (menagerie MJCF + 物体 + 工作台)
  -> Jacobian IK 运动规划 (任务空间原语: reach, grasp, lift, place, push, insert)
  -> MuJoCo 物理仿真批量实验
  -> 指标引擎评估 (接触检测、物体追踪、力监控)
  -> LLM 结果分析 + 下一轮建议
  -> 网页展示指标、图表、报告和轨迹回放
```

## 可行性简析

关键路径为：自然语言解析 → 结构化任务规格 → 动态场景组装 → MuJoCo 批量仿真 → 日志解析 → 下一轮建议。所需资源为 MuJoCo Menagerie 开源机械臂模型、MuJoCo 物理引擎、本地 Python API 与网页 Demo。主要风险是 IK 对某些构型失败和场景组装产生无效 XML，将通过自适应阻尼 IK、XML 验证和降级策略处理。

# 通用机械臂实验智能体实施说明

## 1. 当前目标

本项目已从单一 Demo 展示推进为“通用机械臂实验智能体”的工程化底座。当前版本完成了从自然语言任务、标准实验协议、技能计划、能力匹配、实验设计、执行适配、结果评估到实验记忆的基础闭环。

目标系统的核心链路是：

```text
用户实验目标
  -> TaskSpec
  -> ExperimentPlan
  -> SkillPlan
  -> InstrumentAdapter
  -> EvaluationReport
  -> ExperimentStore
  -> RetryPlan
```

## 4. 当前边界

当前版本已经完成通用实验智能体的软件骨架，但仍有明确边界：

- 真实机械臂驱动尚未接入。
- 复杂任务规划仍是规则和模板驱动，不是通用任务规划器。
- 技能库中部分技能是计划能力，尚未都有物理执行实现。
- 成功率仍基于当前仿真配置和有限样本，不代表跨真实环境泛化。
- 实验记忆是轻量 JSONL，不是完整数据库。

## 4.1 本轮已补强的严格物理与目标约束

针对 Demo 中容易误导的两类问题，当前版本已做同步修正：

- 用户输入“移动 5cm / push 5cm / 0.05m”时，后端会解析为 `target_displacement_m`，并写入 `success_criteria`。
- 推动和工具推动任务不再用“位移超过某个下限”判成功，而是用 `目标位移 ± 容差` 判定；明显推过头会归因为 `overshoot`，没推够会归因为 `undershoot`。
- 桌面高度进入物理安全约束，末端执行器低于桌面安全高度会归因为 `table_penetration`，不能再被算作成功。
- 前端新增“通用机械臂实验智能体闭环”和“严格评估口径”展示，直接显示 TaskSpec、ExperimentPlan、SkillPlan、能力匹配、MuJoCo 执行、EvaluationReport 和 RetryPlan。
- NLP 管线新增一次自动闭环重试：当 EvaluationReport 中出现失败类型时，系统会根据 RetryPlan 生成 revised task spec，自动执行一小轮修正实验，并输出重试前后成功率和原失败类型占比变化。
- NLP 管线新增实验记忆写入：每轮自然语言实验会写入 `results/nlp_experiment_store.jsonl`，保存 TaskSpec、ExperimentPlan、SkillPlan、EvaluationReport、RetryPlan 和 retry execution 摘要。
- 报告中会写出目标位移、容差、样本数、成功数、失败数和物理安全口径。

新增回归验证：

```powershell
scripts\verify_strict_constraints.ps1
```

该脚本固定验证“推动 5cm 且不能陷进桌子”的场景，要求所有成功样本必须同时满足：

- 任务类型被解析为 `push`。
- 目标位移为 `0.05m`。
- 实际位移误差小于容差。
- `table_clearance_ok=true`。

## 5. 下一步建议

后续继续推进时，优先顺序建议如下：

- 将 `/api/nlp/run` 的输出显式包装成 `StandardTaskSpec`。
- 将前端展示的成功率切换为 `EvaluationReport`。
- 将当前 FR3 runner 的技能执行结果继续写入 `ExperimentStore`，并把历史结果用于下一轮实验变量推荐。
- 将 Skill Library 中的计划技能逐步绑定到真实 MuJoCo 执行。
- 增加更多 TaskSpec 示例，覆盖抓取、推拉、工具使用、装配、旋拧。
- 在真实机械臂接入前，继续完善 `RealRobotAdapter` 的安全检查字段。
